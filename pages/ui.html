User interface

There are common patterns, we call them Building Blocks

Designing first of all

There is a .PSD file that contains all the main controls used in Gaia.

You can visit the Wiki page with detailed information about use-cases, interaction, visual design and editable assets.

Coding overview

What can be done?

You can build all the UI for your applications without wirting any line of CSS

Benefits? Full semanthic + accesible structures built in with the latest standards, reviewed and coded for top professionals on the web.

What can't be done?

Building Blocks are just plain .css and .html files, so it does not include any .js, that means if you need some behaviour you need to keep your hands dirty.

But, there is many features CSS only ready in the Building Blocks.

How to...

All is shared

All the resources needed for the Building Blocks are placed in /shared/ folder inside Gaia. This means that the whole system apperance could be changed rewriting these files.
So, you only need to link the required CSS files and append the example HTML.

There are a bunch of Building Blocks, but we can split in three main categories

Dialogs

Action menu
Context menus
Confirm
Dialog boxes with message + accept/dismiss buttons
Edit mode
Edition panels with a dialog-like button toolbar
Status
Notification toasters

Widgets

Headers
Common header bars (title + navigation buttons)
Lists
List style structures, ready for placing floating elements
Scrolling
Additional strcutures for controlling in-app scroll
Tabs
Contains filters and tabs
Drawer
Hidden lateral menu behind app content
Toolbars
Great for commonly used actions
Seekbars
Mainly used for setting range values

Pieces

Buttons
Common buttons
Input areas
Common input areas (e.g. search bars)
Switches
Checkboxes, radio buttons, ON/OFF switches
Progress activity
Common spinners and progress bars
