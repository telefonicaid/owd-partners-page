Domain for hosting your apps

Requires a separate domain

Apps MUST have different domains (subdomains) for each app. You CAN NOT share the same domain/subdomain between apps. You can host your first app in https://app1.mydomain.org/ and the second one in https://app2.mydomain.org/

Use SSL

Using SSL is not expensive this days. Well, it's expensive if you buy those highpriced certs from the internet, but it's not more expensive por CPU-bound. And, if you choose to use SSL, you are proving great things to your app and you really love your users. There are plenty of benefits such as:

  • Users are more protected from sniffing
  • You provide an extra feature to your app

Hosting and how to serve correctly the files

Correct mimetype

You need to serve your files correctly in order to use cool characteristics only allowed in HTML5 apps. It's recommended to serve your files as UTF-8 (and to save plain files to that charset):

  • In Apache: edit your httpd.conf file and add the directive AddDefaultCharset utf-8
  • If you are using Nginx, edit your http directives in nginx.conf adding charset utf-8;

Application Cache (manifest.appcache)

Info Check offline section

You MUST serve your WebApp manifest file always with this in mind:

  • It's a good practise to expire your manifest file, so your users won't have old versions of your app. So add this code (Apache example): ExpiresByType text/cache-manifest "access plus 0 seconds"
  • Your manifest.appcache MUST be served with the mimetype text/cache-manifest

App manifest (manifest.webapp)

Info Check manifest section

This JSON file MUST be served like the application cache:

  • Your manifest.webapp MUST be served with the mimetype application/x-web-app-manifest+json
  • The document is expected to be served as UTF-8
